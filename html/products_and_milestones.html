Financial markets
are going Open Source


Kitanda is an Open Source project that aims to disintermediate the Financial Software industry by offering a modular approach, with a suite of standard financial software components which can replace expensive legacy technology with lean and updated functionality.

Commoditisation decreases costs and allows for a unified infrastructure, in the same way a standardised Apache server is used to run almost all websites in the world.

Dogen

Dogen is a code generator that targets domain models. The user models a domain using a UML tool; Dogen takes this diagram as its input and generates a C++-11 representation. The generated code contains most of the services required from a typical C++ domain (serialisation, hashing, iostreams, etc).
  • Dia - Diagram Editor
  • Domain-driven Design

Architecture

Component Description
Front-end Reads documents generated by a UML UI tool. At present we are only targeting Dia, the Gnome diagram tool.
Middle-end Intermediate representation of the domain model, designed with code generation in mind. We created the Simplified Modeling Language for this, based on ECore and UML but only adding the bare-bones necessary for our needs.
Back-end Outputs code in the supported representations. At present we only support C++-11 but SQL is on our road map. http://www.eclipse.org/modeling/emf/?project=emf http://en.wikipedia.org/wiki/Unified_Modeling_Language

Initial support for reading Dia diagrams
Dogen, the Code Generator, can read Dia diagrams from this point on. One can graphically draw UML diagrams in Dia to express all the required business logic and have it being read by Dogen.
Initial support for C++ code generation
Dogen can output actual C++ code which is graphically created by a Dia UML diagram, thus establishing a full pipeline from drawing to valid code generation. This marks a significant advancement in the architecture, since the code generator will handle a significant amount of code that won't need to be maintained by programmers.
Support for inheritance
Inheritance in UML diagrams is now being analysed for incorporation into Dogen. This will allow the code generator to reuse attributes and behaviour from base classes.
Support for composition
Composition, along with Inheritance, is also being analysed for implementation in Dogen. Composition will stregthen Dogen's abilities by allowing it to combine simple data types into more complex data types.

Dirien

Dirien - the Distributed Risk Engine - is designed to augment quantitative analysis libraries such as QuantLib with all the required facilities needed by production systems: portfolio support, result aggregation, distributed processing (map/reduce), scenario analysis, etc.
  • Quantlib

Architecture

Component Description
Distributed and Map/Reduce Support for a client/server architecture, batching of jobs, etc.
Risk space Scenario analysis with multiple dimensions.
Risk domain model All entities required to model risk such as instruments, portfolios, term structures, results, etc.
Valuator The core of the risk engine, responsible for setting up the pricing engines and associated structures, running the scenarios, aggregating results, etc.
Pricing engine wrapper Defines a set of common APIs for Kitanda to make use of third-party quantitative libraries.
